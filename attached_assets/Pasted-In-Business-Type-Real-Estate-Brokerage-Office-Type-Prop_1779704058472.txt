2. DATABASE SCHEMA (Must implement all tables)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

TABLE: properties
- id, name, type (ENUM: residential/commercial/industrial/land/villa/
  apartment/office/warehouse/mall/compound/hotel/showroom/clinic/other)
- address, city, district, coordinates (lat/lng)

- status (ENUM: available/rented/for_sale/under_maintenance/inactive)
- owner_id, notes, documents (JSON array of file URLs)
- created_at, updated_at

TABLE: units
- id, property_id, unit_number, type, floor
- area_sqm, bedrooms, bathrooms, parking_spaces

TABLE: tenants
- id, full_name, id_number, id_type (national_id/iqama/passport)
- phone, email, nationality, emergency_contact (JSON)
- company_name (if commercial), cr_number
- documents (JSON: id scan, contracts, etc.)
- created_at

TABLE: contracts
- id, unit_id, tenant_id
- start_date, end_date, duration_months
- monthly_rent, total_value, security_deposit
- payment_frequency (ENUM: monthly/quarterly/biannual/annual)
- payment_day (day of month payment is due)
- status (ENUM: draft/active/expired/terminated/renewed)
- auto_renew (boolean), renewal_notice_days
- terms (text), documents (JSON)
- signed_date, terminated_date, termination_reason

TABLE: invoices
- id, contract_id, unit_id, tenant_id
- invoice_number (auto-generated: INV-YYYY-NNNN)
- type (ENUM: rent/deposit/maintenance/utilities/penalty/other)
- amount, tax_amount, total_amount
- due_date, issue_date
- status (ENUM: pending/paid/partial/overdue/cancelled)
- notes

TABLE: payments
- id, invoice_id, contract_id, tenant_id
- amount_paid, payment_date
- method (ENUM: cash/bank_transfer/cheque/online)
- reference_number, bank_name
- received_by (user_id)
- notes, receipt_url

TABLE: expenses
- id, property_id, unit_id (nullable)
- category (ENUM: maintenance/utilities/insurance/tax/management_fee/
  salary/marketing/renovation/legal/other)
- description, amount, tax_amount
- vendor_name, vendor_contact
- expense_date, due_date, paid_date
- status (ENUM: pending/paid/overdue)
- receipt_url, notes

TABLE: maintenance_requests
- id, property_id, unit_id, tenant_id (nullable)
- title, description, category
- priority (ENUM: low/medium/high/urgent)
- status (ENUM: open/assigned/in_progress/completed/cancelled)
- reported_date, scheduled_date, completed_date
- assigned_to (user_id), vendor_name, vendor_contact
- estimated_cost, actual_cost
- before_images (JSON), after_images (JSON), notes

TABLE: journal_entries (double-entry accounting)
- id, entry_number, date, description
- reference_type (invoice/payment/expense/manual)
- reference_id
- created_by (user_id)

TABLE: journal_lines
- id, journal_entry_id
- account_code, account_name
- debit, credit
- notes

TABLE: chart_of_accounts
- id, code, name, type (ENUM: asset/liability/equity/revenue/expense)
- parent_id (for hierarchy), is_active

TABLE: notifications
- id, user_id, type, title, message
- related_type, related_id
- is_read, created_at

TABLE: audit_log
- id, user_id, action, entity_type, entity_id
- old_values (JSON), new_values (JSON), ip_address, created_at

3. BUSINESS LOGIC (Critical — implement exactly)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

A. INVOICE AUTO-GENERATION
- When a contract is created/activated, automatically generate all invoices
  for the entire contract duration based on payment_frequency
- Each invoice has a due_date calculated from payment_day field
- Invoices must include: rent amount + any applicable taxes (VAT 15%)

B. OVERDUE DETECTION
- Run a daily cron job: mark invoices as overdue if due_date < today and status = pending
- Auto-create notifications for overdue invoices
- Apply late penalty if configured (configurable % per day or fixed amount)

C. CONTRACT ALERTS (Automated)
- 90 days before expiry → notification type: "contract_expiring_soon"
- 30 days before expiry → notification type: "contract_expiring_critical"
- Day of expiry → auto-change contract status to "expired"
- Auto-renew if auto_renew = true (create new contract, copy terms, update dates)

D. DOUBLE-ENTRY ACCOUNTING
- Every financial transaction must create journal entries automatically:
  * Payment received → DR: Cash/Bank | CR: Rent Revenue
  * Invoice issued → DR: Accounts Receivable | CR: Rent Revenue
  * Expense paid → DR: Expense Account | CR: Cash/Bank
  * Security deposit → DR: Cash | CR: Security Deposits Liability
- Maintain a full chart of accounts with standard Arabic real estate accounts

E. UNIT STATUS MANAGEMENT
- Unit status must auto-update:
  * Contract activated → unit status = "rented"
  * Contract expired/terminated → unit status = "available"
  * Maintenance request (urgent) → can flag as "under_maintenance"

4. API ENDPOINTS (RESTful)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Implement full CRUD for all entities plus:

GET  /api/dashboard/summary
  → total properties, total units, occupied %, 
    monthly revenue, monthly expenses, net income,
    overdue invoices count and total amount,
    expiring contracts (next 90 days)

GET  /api/reports/income-statement?from=&to=&property_id=
  → Revenue breakdown by property and unit type
  → Expense breakdown by category
  → Net Operating Income (NOI)
  → Gross vs Net yield %

GET  /api/reports/rent-roll
  → All units with: tenant name, rent amount, lease dates,
    payment status, days overdue (if any)

GET  /api/reports/cash-flow?from=&to=
  → Monthly cash in vs cash out, running balance

GET  /api/reports/balance-sheet
  → Assets (property values + receivables + cash)
  → Liabilities (deposits + payables)
  → Owner equity

GET  /api/reports/aging-receivables
  → Outstanding invoices grouped by: current / 30d / 60d / 90d / 90d+

GET  /api/reports/occupancy?from=&to=
  → Occupancy rate per property, per unit type, overall

POST /api/invoices/:id/mark-paid
  → Accept payment details, create payment record, update invoice,
    auto-create journal entries, generate receipt PDF

GET  /api/properties/:id/financial-summary
  → Per-property: total income, total expenses, NOI, yield

POST /api/contracts/:id/terminate
  → Require termination_reason, settlement amount
  → Auto-generate settlement invoice if needed
  → Update unit status, create journal entries
