Completion Score Logic

Compute completion_score (0–100) and status on every save:

score = 0

// 50 pts from 9 required fields (each = 5.56 pts)
required = [company_name, contact_name, phone, cr_number, vat_number,
            bank_name, bank_account_name, iban, city]
score += (filled_required_fields / 9) * 50

// 20 pts from equipment (up to 4 available items, each = 5 pts)
score += min(20, available_equipment_count * 5)

// 30 pts from mandatory documents (each = 10 pts)
mandatory_docs = ['iban_cert', 'cr_cert', 'vat_cert']
score += uploaded_mandatory_docs * 10

status = score == 100 ? 'complete' : score > 0 ? 'partial' : 'draft'


Supplier Performance Score (for Reports/Rankings)

score = 0

// 1. Completion (0–30)
score += completion_score * 0.30

// 2. Equipment breadth (0–20)
available_eq = count of available equipment
score += min(20, (available_eq / total_eq_types) * 20)

// 3. Pricing competitiveness (0–25)
// Has at least one equipment with a daily_rate set = 25 pts
if (any equipment has daily_rate > 0): score += 25

// 4. Documents (0–15): 3 pts per uploaded doc (5 slots max)
score += uploaded_doc_count * 3

// 5. Payments (0–10): ratio of paid installments
if (total_installments > 0): score += (paid_installments / total_installments) * 10

POST   /api/suppliers/:id/documents      -- upload doc (multer, field: doc_key)

GET    /api/suppliers/alerts             -- overdue/due-soon payments + ending rentals

All endpoints require existing BSS auth middleware. Return JSON. Use existing BSS error handling pattern.
